# CONTEXT — provenance & spine

**This file is background, not for publication.** It captures what the
post(s) are *about*, why this argument matters, and what the load-bearing
distinctions are. The publishing agent reads this to understand the
spine; the published artifacts don't include it.

## Goal & scope

- What question or topic does this bundle address?
- Two-level frame: the immediate problem, and the larger system it
  lives inside. (Per the project's gate: both must be coherent before
  proceeding.)
- Who is the audience — general readers, domain practitioners, or both?

## Language / glossary

Define the load-bearing terms once. The publishing agent should use
these definitions consistently across the short and long artifacts.

- **Term A** — definition. *Avoid* using "term A" to mean (X) — that
  collapses a distinction we care about.
- **Term B** — definition.

## The spine

State the single structural claim of the bundle in 1–2 sentences. This
is the load-bearing assertion the artifacts have to land. If a reviewer
forgets everything else, they should remember this.

## Why this framing (resolved positions)

- Positions that were considered and rejected, with why.
- Distinctions the author considers settled — these are the ones
  publishing should not relitigate.

## Open questions

- Things still genuinely open. The artifacts should not pretend these
  are resolved.

## Suggested arc

How the artifact should unfold for the reader (act I / act II / act III,
or whatever the natural structure is). The publishing agent uses this
to decide section ordering, figure placement, and where to surface the
spine.
